# Haski Client Bot - Полная документация

## 1) Что это за проект

Telegram-бот магазина `Haski Client` на `aiogram` + `SQLite`.

В проекте есть:
- пользовательское меню (`купить`, `профиль`, `поддержка`);
- админ-панель (`/admin`) с управлением товарами;
- рассылка всем пользователям;
- очередь обращений в поддержку с ответом/игнором;
- два способа покупки из карточки товара:
1. ссылка на FunPay;
2. ссылка на личные сообщения (`t.me/<username>`).


## 2) Быстрый ответ на ваш вопрос (FunPay + username)

Где хранится ссылка FunPay:
- файл `.env`
- переменная: `FUNPAY_URL=...`

Где указывается username для покупки через личные сообщения:
- файл `.env`
- переменная: `PM_USERNAME=...` (можно с `@`, например `@haski_support`)

Как это подключается в коде:
- `config.py` читает переменные окружения (`FUNPAY_URL`, `PM_USERNAME`);
- `keyboards/inline.py` использует их при построении кнопок покупки.


## 3) Стек и зависимости

- Python 3.10+
- `aiogram` (бот + роутинг + FSM)
- `python-dotenv` (чтение `.env`)
- `sqlite3` (встроен в Python)

Минимальная установка:

```bash
pip install aiogram python-dotenv
```


## 4) Запуск проекта

1. Заполнить `.env`:

```env
TOKEN=...
ADMINS='123456789'
DB_PATH=databases/shop_data.db
FUNPAY_URL=https://funpay.com/
PM_USERNAME=@your_username
```

2. Запустить:

```bash
python main.py
```

При старте бот:
1. создает/проверяет таблицы БД;
2. регистрирует роутеры;
3. запускает polling Telegram.


## 5) Структура проекта

```text
Haski_Client/
```

### Назначение файлов

- `main.py` - точка входа, запуск бота.
- `config.py` - чтение настроек из `.env`.
- `databases/queries.py` - все SQL-операции.
- `filters/admin_filter.py` - проверка, что пользователь админ.
- `handlers/start.py` - обработка `/start`.
- `handlers/menu.py` - пользовательский интерфейс.
- `handlers/admin.py` - админ-панель и сценарии админа.
- `keyboards/inline.py` - inline-кнопки пользовательской части.
- `keyboards/admin_kb.py` - inline-кнопки админ-панели.
- `states/*.py` - состояния FSM для пошаговых сценариев.


## 6) Переменные окружения (`.env`)

- `TOKEN` - токен Telegram-бота.
- `ADMINS` - ID админов через запятую.
  - пример: `ADMINS='111111111,222222222'`
- `DB_PATH` - основной путь к SQLite-файлу.
- `FUNPAY_URL` - ссылка на FunPay для кнопки покупки.
- `PM_USERNAME` - username для кнопки ЛС.


## 7) Подробная логика по коду

## 7.1 Старт и регистрация пользователя

Файл: `handlers/start.py`

Сценарий `/start`:
1. Пользователь добавляется/обновляется в таблице `users`.
2. Отправляется главное меню с кнопками:
   - купить;
   - профиль;
   - поддержка.


## 7.2 Пользовательское меню

Файл: `handlers/menu.py`

### Профиль
- Кнопка `profile` показывает:
  - username;
  - заглушки баланса/подписки.

### Каталог (`store`)
1. Загружаются активные товары из БД.
2. Показ списка кнопок вида: `Название - Цена RUB`.
3. По нажатию на товар открывается карточка:
   - название;
   - цена;
   - описание;
   - кнопки покупки.

### Покупка товара
- Кнопка `FunPay` ведет на `FUNPAY_URL`.
- Кнопка `Личные сообщения`:
  - если задан `PM_USERNAME`, открывает `https://t.me/<username>`;
  - если не задан, показывает подсказку с тем, что username не настроен.

### Поддержка
1. Пользователь нажимает `поддержка`.
2. Нажимает `написать в поддержку`.
3. Вводит текст.
4. Бот создает запись в `support_tickets` со статусом `open`.
5. Пользователь получает номер обращения.


## 7.3 Админ-панель

Файл: `handlers/admin.py`

Вход в панель: команда `/admin`.

Панель доступна только пользователям, прошедшим `AdminFilter`:
- фильтр в `filters/admin_filter.py`;
- сравнивает `from_user.id` со списком `ADMIN_IDS` из `.env`.

### Управление товарами

#### Добавление
FSM `AddProduct`:
1. ввести название;
2. ввести цену (только положительное целое);
3. ввести описание;
4. сохранить в таблицу `products`.

#### Редактирование
FSM `EditProduct`:
1. выбрать товар;
2. ввести новую цену;
3. обновить цену в БД.

#### Удаление
1. выбрать товар;
2. удалить запись из `products`.


### Рассылка
FSM `BroadcastMessage`:
1. админ вводит текст;
2. бот берет `user_id` из таблицы `users`;
3. отправляет сообщение каждому;
4. возвращает статистику `доставлено/ошибок`.


### Поддержка (очередь админа)
FSM `SupportReply`:
1. админ открывает раздел обращений;
2. листает очередь (`prev/next`);
3. может:
   - `ответить` - отправить ответ пользователю и закрыть тикет (`answered`);
   - `игнор` - пометить как `ignored`.

После ответа бот отправляет пользователю сообщение с его исходным текстом и ответом поддержки.


## 8) База данных

Файл: `databases/queries.py`

Создаются 3 таблицы:

### `products`
- `id` INTEGER PK
- `name` TEXT
- `price` INTEGER
- `description` TEXT
- `is_active` INTEGER DEFAULT 1

### `users`
- `user_id` INTEGER PK
- `username` TEXT
- `full_name` TEXT

### `support_tickets`
- `id` INTEGER PK
- `user_id` INTEGER
- `username` TEXT
- `full_name` TEXT
- `message` TEXT
- `status` TEXT (`open` / `answered` / `ignored`)
- `admin_response` TEXT
- `created_at` TEXT
- `answered_at` TEXT


## 9) Особенность работы с БД (OneDrive-safe fallback)

В `queries.py` реализован выбор рабочего пути БД:
1. сначала пробуется `DB_PATH` из `.env`;
2. если путь не writable (частая проблема синхронизируемых папок), используется fallback в temp-папке системы.

Это снижает риск падения бота из-за блокировок SQLite в cloud-sync директориях.


## 10) Безопасность и корректность

Что уже сделано правильно:
- админ-функции защищены `AdminFilter`;
- обычный пользователь не должен попасть в админ-роуты;
- SQL-запросы параметризованы (`?`), снижает риск SQL-инъекций;
- пользовательские тексты в HTML-сообщениях экранируются через `html.escape`.

Что важно соблюдать:
- хранить `TOKEN` приватно;
- регулярно проверять `ADMINS` в `.env`;
- не отправлять в репозиторий рабочие файлы БД.


## 11) Почему может быть «кракозябрами» текст

Если в Telegram отображается непонятный текст вида `РџС...`, это проблема кодировки файла (обычно смесь UTF-8/CP1251).

Проверить:
1. все `.py` должны быть сохранены в UTF-8;
2. редактор/IDE должен открывать и сохранять файлы в UTF-8 без BOM (или единообразно UTF-8);
3. после исправления перезапустить бота.
